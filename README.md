    idle->Pressed->Dragging->Dropped->Idle

<div>
  <h2 align="center">Порогові значення для double-click і long-press</h2>
  <dl>
    <dt>Double-click:</dt>
    <dd>
      Зазвичай у UI прийнято 200–300 мс між кліками.
      <ul>
        <li>Якщо менше — користувач не встигає.</li>
        <li>Якщо більше — система може сприйняти це як два окремі кліки.</li>
      </ul>
      <mark>Я б запропонував 250 мс як баланс між швидкістю і комфортом</mark>
    </dd>
    <dt>Long-press:</dt>
    <dd>
      У мобільних і десктопних інтерфейсах стандарт — 500–700 мс.
      <mark>Оптимально 600 мс: достатньо довго, щоб відрізнити від звичайного кліку, але не надто довго, щоб користувач не чекав.</mark>
    </dd>
  </dl>
</div>
<div>
  <h2 align="center">Debounce vs Throttle</h2>
  <dl>
    <dt>Debounce (відсікання зайвих викликів):</dt>
    <dd>
        Використати для обробки клавіш у keychecker().
        <ul>
          <li>Якщо користувач утримує клавішу, функція викликається кожен кадр (draw()), що може бути занадто швидко.</li>
          <li>Debounce дозволить реагувати лише після паузи між натисканнями (наприклад, щоб не рухати гравця занадто різко при швидкому кліку).</li>
        </ul>
    </dd>
    <dt>Throttle (обмеження частоти викликів):</dt>
    <dd>
        Використати для dragging.
        <ul>
          <li>Коли мишу тягнуть, mouseDragged може викликатися десятки разів на секунду.</li>
          <li>Throttle (наприклад, раз на 50 мс) зменшить навантаження і зробить рух більш стабільним, особливо при складних обчисленнях чи рендерингу.</li>
        </ul>
    </dd>
  </dl>
</div>
